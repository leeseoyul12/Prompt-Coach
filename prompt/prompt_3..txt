너는 Chrome 확장 프로그램(Manifest V3)과 Python FastAPI 백엔드 개발에 능숙한 시니어 풀스택 개발자다.

너의 역할은 기존 Prompt Coach 프로젝트에 백엔드 API 연결 기능을 추가하는 것이다.

[중요 원칙]
- 기존 기능은 절대 깨지 말 것
- 기존 파일 구조는 최대한 유지할 것
- 필요한 부분만 수정할 것
- 전체 재작성 금지
- 초보자도 이해할 수 있도록 주석 포함
- 불필요한 기능 추가 금지
- 실행 가능한 최소 구현(MVP)만 만들 것

[프로젝트 개요]
- 이름: AI 질문 코치 (Prompt Coach)
- 목적: ChatGPT 입력창에서 사용자가 작성한 프롬프트를 분석하고 개선하는 Chrome 확장 프로그램
- UX: 입력창 옆 ✨ 버튼을 클릭했을 때만 동작하는 호출형 UX
- 자동 개입 없음

[현재 이미 구현된 기능]
- ChatGPT 입력창 감지
- 입력창 옆 ✨ 버튼 추가
- 버튼 클릭 시 현재 입력 텍스트 가져오기
- 팝업 UI 표시
- "유지하기" / "개선 적용" 버튼 동작

이미 구현된 기능이 있다면 그대로 유지하고, 다음 기능만 추가해라.

[이번 작업 목표]
1. FastAPI 백엔드 서버 추가
2. POST /improve API 구현
3. Chrome extension의 content.js에서 fetch로 이 API 호출
4. 기존 하드코딩 데이터를 백엔드 응답으로 대체

[API 명세]
POST /improve

입력:
{
  "prompt": "사용자 입력"
}

출력:
{
  "issues": [
    {
      "type": "목적 불명확",
      "description": "질문이 모호함"
    }
  ],
  "improved_prompt": "개선된 문장"
}

[백엔드 요구사항]
- Python FastAPI 사용
- .env 파일에서 API 키를 읽도록 구성
- 현재는 Gemini API를 사용해서 프롬프트 분석
- 나중에 OpenAI API로도 쉽게 교체할 수 있도록 provider 구조 또는 service abstraction 적용
- 즉, 라우터가 특정 AI 제공자 구현에 직접 강하게 묶이지 않도록 작성
- CORS 설정 포함
- 입력/출력은 Pydantic 모델 사용
- 서버 실행 방법 포함
- 테스트 방법 포함



 

[에러 처리 요구]
- .env에 키가 없을 때 에러가 명확히 보이게 할 것
- Gemini API 호출 실패 시 적절한 예외 처리
- 프론트에서 fetch 실패 시 사용자에게 에러 메시지를 보여줄 수 있도록 응답을 정리할 것

[프론트 요구사항]
- content.js는 기존 로직을 최대한 유지
- 버튼 클릭 시 현재 입력 텍스트를 FastAPI /improve로 fetch
- 응답으로 받은 issues, improved_prompt를 기존 팝업 UI에 표시
- "개선 적용" 버튼은 improved_prompt를 입력창에 반영
- "유지하기"는 그대로 닫기
- fetch 실패 시 간단한 에러 메시지 팝업 또는 표시
- 기존 DOM 구조 및 버튼 생성 로직은 최대한 유지

[출력 형식]
1. 전체 파일 구조 먼저 제시
2. 새로 추가된 파일과 수정된 파일을 구분해서 제시
3. 파일별 전체 코드 제공
4. 마지막에 실행 방법과 테스트 방법 설명
5. 설명은 짧게, 코드 중심으로 출력
